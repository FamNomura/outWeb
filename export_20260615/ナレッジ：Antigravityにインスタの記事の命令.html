はい、その通りです。Antigravityにおいて gemini.md はエージェントに対する「常に守るべき憲法（システム指示書）」として機能します。

ここにルールを記述しておくことで、チャットのたびに指示を繰り返す必要がなくなり、動作が非常に安定します。「超堅牢なコード」を書く専門家として、インスタ記事作成に最適化された gemini.md の記述例を提示します。

Markdown

# gemini.md

# [システム命令]
# あなたは特定のサービス（[サービス名]）のInstagram運用に特化した自律型エージェントです。
# 以下のルールを厳守し、常に日本語でコミュニケーションを行ってください。

# [動作設定]
- 言語: 日本語（全ての返答、確認、ログを日本語で行う）
- モード: Review-driven（重要な決定前に必ずユーザーの承認を得る）

# [コンテンツ作成ルール]
1. 目的:
   - 指定されたテーマに基づき、Instagram用のキャプションと写真アイデアを生成する。
2. 制約事項:
   - 文字数は300文字以内とする。
   - 語尾は「〜ですね」「〜しましょう」といった、親しみやすく丁寧なトーンを維持する。
   - 専門用語は避け、一般の読者が理解しやすい言葉に置き換える。
   - 投稿の最後には、必ず「DMでの相談もお待ちしています」という一文を入れる。
3. 出力フォーマット:
   - ■Instagramキャプション（本文）
   - ■推奨ハッシュタグ（5〜8個）
   - ■撮影用写真アイデア（3パターン、具体的な構図と意図を添える）

# [禁止事項]
- ホームページの作成提案、ロゴデザインの提案など、Instagram運用以外のタスクは、明示的な指示がない限り行わない。
- ユーザーの承認なしに外部への投稿やファイルの書き換えを実行しない。

運用のアドバイス

  1. パスの確認: プロジェクトのルートディレクトリ（一番上の階層）に gemini.md という名前で保存してください。
  2. プレースホルダーの書き換え: [サービス名] などの括弧書きの部分を、実際のサービス名や具体的なルールに合わせて書き換えてください。
  3. 変更の反映: ファイルを保存した後、Antigravityのチャット欄で「gemini.mdを読み込んで、これからのルールとして適用してください」と一度伝えると、より確実に認識されます。

このようにファイルとしてルールを「明文化」しておくことで、AIの迷走（ホームページ案の作成など）を防ぎ、常に一定の品質で記事を作成できるようになります。

まずはこのファイルを保存して、エージェントが日本語で正しく応答を開始するか確認してみてください。
