ご質問への回答と技術的な実態

楽天APIの利用とレビュー全件取得について

APIの利用: はい、楽天は「楽天ウェブサービス（商品検索APIなど）」を無償で提供しており、価格、アフィリエイトリンク、レビュー件数、平均評価点などはAPIで安全かつ高速に取得可能です。

レビュー本文の取得: 残念ながら、楽天APIでは「レビューの本文」を取得することはできません。 そのため、楽天に関しても「商品の検索と基本情報の取得はAPIで行い、レビュー内容の分析が必要な場合は対象URLをスクレイピングする」というハイブリッドな設計が必要です。

Amazonの「レビュー分析」について

Amazonが商品ページ上部に表示しているAIによる「お客様の声（Review Highlights）」をスクレイピングで抽出するアプローチは、データ量を抑えつつ的確な情報を得る上で非常に効率的です。ただし、この部分は動的に生成されることが多く、HTMLの構造（DOM）が頻繁に変更されるため、プログラム側でエラー検知と柔軟な要素指定（セレクタの更新）を行う堅牢な設計が求められます。

X（Twitter）自動投稿のコストとプラグインの制限

X APIの無料枠: X（旧Twitter）が公式に提供している無料プラン（Free tier）では、「1ヶ月あたり1,500件（1日約50件）」までの投稿が可能です。初期の運用であれば十分な枠です。

WordPressプラグインの実態: 注意点として、多くの連携用WPプラグイン（Jetpack Socialなど）は、プラグイン開発元がXに高額なAPI利用料を払っているため、ユーザーに対する「無料枠」を非常に少なく設定しています（例：月に30回まで等）。

解決策: コストを完全に抑え、かつ確実な動作を保証するためには、WPプラグインに頼らず、「WordPressのRSSフィードを更新トリガーとし、自動化ツール（MakeやZapierの無料枠）を経由して、ご自身で取得したXの無料APIキーを使って投稿する」という設計が最もコントロールしやすく堅牢です。

1. 設計の基本方針

追跡可能性: すべてのデータに取得日時（created_at）を記録し、価格推移のグラフ化や投稿履歴の監査を容易にする。

import sqlite3

def initialize_database():

    cursor = conn.cursor()

    # テーブル作成: items (商品マスタ)

    cursor.execute('''

    CREATE TABLE IF NOT EXISTS items (

        id INTEGER PRIMARY KEY AUTOINCREMENT,

        platform TEXT NOT NULL,                -- 'amazon' or 'rakuten'

        platform_product_id TEXT NOT NULL,      -- ASIN or JAN

        title TEXT NOT NULL,

        category TEXT,

        url TEXT,

        is_active INTEGER DEFAULT 1,            -- 0: 停止, 1: 稼働

        created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

        UNIQUE(platform, platform_product_id)   -- 同一プラットフォームでの重複防止

    )

    ''')

    # テーブル作成: price_history (価格履歴)

    cursor.execute('''

    CREATE TABLE IF NOT EXISTS price_history (

        id INTEGER PRIMARY KEY AUTOINCREMENT,

        item_id INTEGER NOT NULL,

        price INTEGER NOT NULL,

        is_sale INTEGER DEFAULT 0,

        fetched_at DATETIME DEFAULT CURRENT_TIMESTAMP,

        FOREIGN KEY (item_id) REFERENCES items(id)

    )

    ''')

    # テーブル作成: review_analysis (レビュー分析)

    cursor.execute('''

    CREATE TABLE IF NOT EXISTS review_analysis (

        id INTEGER PRIMARY KEY AUTOINCREMENT,

        item_id INTEGER NOT NULL,

        rating REAL,

        review_count INTEGER,

        summary_text TEXT,

        analyzed_at DATETIME DEFAULT CURRENT_TIMESTAMP,

        FOREIGN KEY (item_id) REFERENCES items(id)

    )

    ''')

    # テーブル作成: posts (投稿履歴)

    cursor.execute('''

    CREATE TABLE IF NOT EXISTS posts (

        id INTEGER PRIMARY KEY AUTOINCREMENT,

        item_id INTEGER NOT NULL,

        wp_post_id INTEGER,                     -- WordPressの投稿ID

        post_status TEXT DEFAULT 'draft',       -- 'draft', 'publish'

        post_url TEXT,

        tweet_id TEXT,                          -- XのポストID

        last_posted_at DATETIME,

        FOREIGN KEY (item_id) REFERENCES items(id)

    )

    ''')

    # テーブル作成: item_metadata (拡張メタデータ)

    cursor.execute('''

    CREATE TABLE IF NOT EXISTS item_metadata (

        id INTEGER PRIMARY KEY AUTOINCREMENT,

        item_id INTEGER NOT NULL,

        meta_key TEXT NOT NULL,                 -- 'local_image_path', 'resale_value' 等

        meta_value TEXT,

        updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,

        FOREIGN KEY (item_id) REFERENCES items(id)

    )

    ''')

    conn.commit()

    conn.close()

    print(f"Database initialized at: {db_path}")
